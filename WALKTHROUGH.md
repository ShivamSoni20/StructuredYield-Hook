# StructuredYield Walkthrough

Last updated: June 7, 2026

## Project Summary

StructuredYield is a Uniswap V4 hook project that turns LP positions into fixed-income style primitives:

- **PT-LP**: principal token for fixed principal redemption at maturity.
- **YT-LP**: yield token for the fee stream generated by the LP position.
- **Insurance vault**: scaffold reserve accounting for impermanent-loss coverage.
- **Periphery**: router/lens helpers for demo-friendly frontend reads and actions.

The project now has contracts, tests, deployment scripts, a component-based Next.js frontend, demo fallback data, subgraph scaffolding, and deployment documentation.

## What Works Now

### Contract Scaffold

Working in the dependency-light scaffold:

- Create pool.
- Mint PT/YT through `SYRouter.depositAndMint`.
- Route/account fees through hook fee accounting.
- Claim YT fees through `SYRouter.claimFees`.
- Redeem after maturity through `SYRouter.removeAndRedeem`.
- Read position data through `SYLens.getPosition`.

### Frontend

Working:

- Landing page renders as real Next.js UI.
- Wallet connect button uses RainbowKit.
- Connected wallet redirects to dashboard.
- Dashboard route is protected from disconnected users.
- Dashboard and position pages show demo fallback data.
- Deposit, claim, and redeem buttons are wired to Wagmi hooks.
- Markets and positions pages are available.
- Mobile dashboard navigation moves to the bottom.

### Demo Mode

When deployed contracts are unavailable or reads fail, the frontend uses demo positions:

- `ETH / USDC`
- `wBTC / USDC`
- `wstETH / USDC`

This lets the UI remain demoable without requiring all contracts/subgraph data to be live.

## Swap Status

### Are real Uniswap V4 swaps fully working?

**Not fully yet.**

The project has hook/accounting logic for `afterSwap`, and the V4 adapter implements the `afterSwap` callback surface. However, a production-grade live swap path through a real V4 `PoolManager` is **not fully verified** yet.

Current status:

- Scaffold fee routing works through direct hook/accounting calls and tests.
- The V4 adapter exists, but real PoolManager settlement and swap testing still need final integration.
- `StructuredYieldV4Hook` must be deployed to an address with correct Uniswap V4 hook permission bits.
- `MineHookAddress.s.sol` documents the required CREATE2 permission-bit mining helper path.
- The frontend does not yet perform live Uniswap swaps for YT-LP; trade UI currently communicates demo/scaffold status.

### Can Uniswap route through this hook natively?

**No for the current scaffold submission.**

Reasons:

- The router uses direct hook calls in scaffold mode.
- V4 hook address permission-bit mining is documented but not completed as a production deployment.
- The current demo is dependency-light and not a finalized PoolManager-native liquidity/swap path.

## Known Limitations

- `InsuranceVault` currently uses accounting reserves and does not custody real USDC/WETH/ETH.
- Real token custody must be added to `fund()` and `payout()` before production use.
- Live V4 hook deployment needs CREATE2 address mining for permission bits.
- YT transferability needs more fee ownership/snapshot work before production.
- `sqrtPrice` is still passed through scaffold/demo paths instead of fully pulled from PoolManager state everywhere.
- Subgraph deployment still needs the final deployed contract address and deployment block.
- Slither and Foundry should be rerun in an environment with all tools installed before final submission.

## How To Run Locally

Frontend:

```bash
cd frontend
npm install
npm run dev -- -p 3000
```

Open:

```text
http://localhost:3000
```

Contracts:

```bash
cd contracts
forge test -vvv
```

Deploy to Unichain Sepolia:

```bash
cd contracts
cp .env.example .env
# Fill PRIVATE_KEY in .env
source .env

forge script script/Deploy.s.sol \
  --rpc-url unichain_sepolia \
  --broadcast \
  --verify \
  --verifier blockscout \
  --verifier-url https://unichain-sepolia.blockscout.com/api \
  -vvvv
```

Create demo pool:

```bash
STRUCTURED_YIELD_HOOK=<deployed_hook_address> \
forge script script/CreatePool.s.sol \
  --rpc-url unichain_sepolia \
  --broadcast \
  -vvvv
```
